# 里程碑、风险与取舍

## 1. 里程碑总览

| 阶段 | 周期 | 目标 |
|---|---:|---|
| P0 架构骨架 | 2 周 | daemon、CLI、元数据、任务队列、基础 API |
| P1 文本导入与全文检索 | 3-4 周 | docx/PDF 文本层、倒排索引、字段过滤 MVP |
| P2 字段抽取与去重 | 3-4 周 | 联系方式、教育、工作、技能、证书、候选人折叠 |
| P3 语义检索 | 3-4 周 | embedding、向量索引、混合检索、RRF |
| P4 OCR 支路 | 3-4 周 | 扫描 PDF 检测、OCR worker、缓存、限流 |
| P5 跨平台打包 | 2-3 周 | Windows/macOS 安装、签名、升级、卸载 |
| P6 性能压测和稳定性 | 持续 | 10 万/100 万压测、回归门禁、故障注入 |

如果只有 1 名强工程师，不要并行铺太多功能。先把“可导入、可查询、可恢复”做扎实。

## 2. P0：架构骨架

交付：

1. Rust workspace。
2. daemon 启停。
3. CLI status/search/import skeleton。
4. 元数据库 schema v1。
5. 任务队列。
6. 本地 IPC skeleton。
7. 日志和诊断 skeleton。
8. CI PR workflow。

退出标准：

1. Windows/macOS 均可启动。
2. 能创建数据目录。
3. 能提交导入任务。
4. 能查询状态。
5. kill 后任务状态可恢复。

## 3. P1：文本导入与全文检索

交付：

1. 文件扫描。
2. docx 文本抽取。
3. PDF 文本层抽取。
4. 文本清洗。
5. 基础分段。
6. 全文索引。
7. 文件名和全文查询。
8. snippet。

退出标准：

1. 10 万文本简历可导入。
2. 关键词查询可用。
3. 不支持 OCR 的扫描件会进入 `OCR_REQUIRED`，不会卡死。

## 4. P2：字段抽取与去重

交付：

1. 邮箱、手机、姓名候选。
2. 学校、学历、专业。
3. 公司、职位、时间段。
4. 技能词典。
5. 证书词典。
6. 字段置信度。
7. 候选人软去重。
8. 字段过滤查询。

退出标准：

1. 基础字段 F1 达到 MVP 目标。
2. 能按学历、技能、年限过滤。
3. 同一候选人多版本可折叠。

## 5. P3：语义检索

交付：

1. embedding 模型 ONNX 化。
2. batch 推理。
3. 向量索引。
4. 文档级向量。
5. section 级向量可选。
6. 混合检索。
7. RRF 融合。
8. 查询质量基准。

退出标准：

1. 语义查询可召回同义表达。
2. 向量生成可暂停。
3. 向量索引损坏时全文查询不受影响。

## 6. P4：OCR 支路

交付：

1. PDF 文本层判定。
2. 扫描页检测。
3. OCR worker。
4. OCR 缓存。
5. 页级超时。
6. OCR 语言 profile。
7. OCR 结果入分段和字段抽取。

退出标准：

1. 扫描 PDF 可进入可搜索状态。
2. OCR 任务不会阻塞查询。
3. 低配机器可关闭 OCR。
4. OCR 中断后可重试。

## 7. P5：跨平台打包

交付：

1. Windows MSI。
2. macOS pkg/dmg。
3. 签名。
4. 升级测试。
5. 卸载测试。
6. user-mode daemon。
7. 数据目录迁移。

退出标准：

1. 两个平台均能安装、导入、查询、升级、卸载。
2. 升级不删除用户数据。
3. 回滚路径可验证。

## 8. P6：性能和稳定性

交付：

1. 10 万夜间压测。
2. 100 万发布压测。
3. 性能 regression 门禁。
4. 故障注入。
5. 诊断包。
6. runbook。

退出标准：

1. 百万级热查询达到目标区间。
2. 导入中查询仍可用。
3. 随机 kill 不破坏已提交快照。

## 9. 主要风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| PDF 格式复杂 | 解析质量差 | 多 parser fallback、OCR 判定、质量分 |
| OCR 太慢 | 低配机器卡顿 | 默认限流、可关闭、页级缓存 |
| 字段抽取误差 | 结构化过滤不准 | 置信度、原文证据、人工反馈闭环 |
| 向量内存过大 | 百万级内存压力 | int8/fp16、分片、FAISS/PQ 备选 |
| 索引损坏 | 无法查询 | 快照、checksum、回滚、重建 |
| Windows 环境差异 | 安装和文件锁问题 | user-mode daemon、长路径测试、重试 |
| macOS 权限 | 文件访问受限 | 清晰权限提示、LaunchAgent |
| 模型许可 | 商用风险 | 模型卡审查、可替换模型 |
| 日志泄露 PII | 合规风险 | 脱敏库、CI 检查、诊断包红线 |

## 10. 明确取舍

| 取舍 | 选择 |
|---|---|
| 原型速度 vs 最终性能 | 最终性能 |
| 单库全能 vs 分层索引 | 分层索引 |
| 默认 OCR 全开 vs 按需 OCR | 按需 OCR |
| 大模型端到端解析 vs 规则+小模型 | 规则+小模型 |
| 服务化数据库 vs 嵌入式内核 | 嵌入式内核 |
| 查询完整性 vs 低延迟 | partial 结果优先 |
| 自动强去重 vs 保留多版本 | 保留多版本，软折叠 |

## 11. 后续演进

稳定后再考虑：

1. 更强 reranker。
2. 复杂 query rewrite。
3. 行业专用技能图谱。
4. 候选人推荐解释。
5. 局域网团队版。
6. 增量学习字段修正。
7. 更高压缩率向量索引。

不要在 MVP 阶段做这些。
