# 数据模型与存储协议

## 1. Manifest 主模型

Manifest 是下一阶段的根数据模型。

| 字段 | 作用 | 说明 |
|---|---|---|
| `file_pk` | 内部主键 | 单机递增 |
| `stable_file_id` | 稳定文件身份 | Windows 使用 volume + file id；macOS/POSIX 使用 dev + ino + birthtime 辅助 |
| `root_id` | 来源 root | root 可 offline、disabled、rescanning |
| `display_path` | 用户展示路径 | 仅本地显示，不进入公开证据 |
| `canonical_path_hash` | 路径哈希 | 诊断和匹配用 hash，不提交 path |
| `size_bytes` | 文件大小 | quick diff |
| `mtime_ns` | 修改时间 | quick diff |
| `birthtime_ns` | 创建时间 | inode/fileid 回收防护 |
| `quick_fingerprint` | 快速内容指纹 | 判断是否值得解析 |
| `strong_hash` | 强内容哈希 | 按需计算，支持跨路径识别 |
| `file_kind` | 文件类型 | pdf/docx/doc/txt/image/unsupported |
| `parse_state` | 解析状态 | state machine 输入 |
| `text_epoch` | 文本可见 epoch | 文本层搜索可用性 |
| `fulltext_epoch` | 全文索引 epoch | BM25 可用性 |
| `field_epoch` | 字段索引 epoch | filter 可用性 |
| `vector_epoch` | 向量 epoch | semantic 可用性 |
| `snippet_epoch` | snippet sidecar epoch | 结果片段可用性 |
| `last_seen_scan_epoch` | 最近扫描轮次 | 删除和 offline 判断 |
| `tombstone` | 逻辑删除 | 查询默认隐藏 |
| `retry_after_ts` | 重试时间 | 退避 |
| `error_code` | 最近错误码 | diagnostics 和 GUI 展示 |
| `parser_version` | 解析器版本 | 版本变更触发局部重解析 |
| `schema_version` | 存储/索引 schema | breaking change 门槛 |

## 2. Mutation Batch

所有导入和增量更新都必须经过 batch 协议：

```text
prepare(batch_id, root_id, manifest_delta, target_epoch)
apply_fulltext(batch_id)
apply_fields(batch_id)
apply_snippet_sidecar(batch_id)
apply_vector(batch_id)
commit(batch_id, visible_epoch, commit_ids)
gc(compactable_before_epoch)
```

## 3. 恢复规则

| 状态 | 恢复动作 |
|---|---|
| prepared 但未 apply | 重放 batch |
| fulltext 已 apply 但 SQLite 未 commit | 用 `batch_id` 幂等对账 |
| SQLite 已 commit 但派生层缺失 | 从 batch log 补写派生层 |
| vector 失败 | semantic partial，keyword/field 仍可用 |
| snippet sidecar 缺失 | 返回无 snippet 或短 fallback，不阻塞搜索 |
| tombstone 长期存在 | 等旧 reader drain 后物理清理 |

## 4. Visible Epoch

Visible epoch 是 GUI、CLI、daemon 和 Codex 验证共同理解系统状态的唯一版本点。

每个 response 至少能说明：

1. `visible_epoch`
2. `fulltext_epoch`
3. `field_epoch`
4. `vector_epoch`
5. `snippet_epoch`
6. `partial_reason`

## 5. Sidecar

正文和长 snippet 不进入 Tantivy 热路径 STORED 大字段。

| Sidecar | 内容 | 访问方式 |
|---|---|---|
| snippet sidecar | top result 片段、短摘要、highlight window | doc id -> pointer -> chunk |
| body sidecar | 规范化正文和 section chunk | detail 或 snippet fallback 才读 |
| vector sidecar | embedding payload、ANN snapshot、tombstone | semantic layer 读取 |

写入采用同卷 temp + 校验 + atomic replace/rename。
