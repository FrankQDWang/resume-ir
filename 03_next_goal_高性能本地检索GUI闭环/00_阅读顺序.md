# 阅读顺序

本目录描述下一阶段 Codex 目标：把 `resume-ir` 从 current-stage 可用闭环推进到高性能本地检索、GUI/manual usage 和 Codex 结对验证闭环。

建议阅读顺序：

1. `01_目标边界与成功标准.md`
2. `02_系统架构与模块边界.md`
3. `03_数据模型与存储协议.md`
4. `04_数据流与状态机.md`
5. `05_Query_Benchmark与真实Query种子.md`
6. `06_Daemon_IPC与Diagnostics契约.md`
7. `07_GUI与手工Codex闭环.md`
8. `08_失败模式与恢复策略.md`
9. `09_安全隐私与本地证据边界.md`
10. `10_实施切片与验收门槛.md`
11. `11_一页版目标图.md`
12. `12_Review问题映射与修复责任.md`
13. `13_Loop_Engineering状态机.md`
14. `14_W0_W1验收矩阵与证据协议.md`
15. `15_性能观测与Profiling工具链.md`
16. `16_跨平台全盘发现与增量Journal契约.md`
17. `17_机器可读Goal与Experiment协议.md`

总原则：**先稳定观察面，再改数据面；先热路径可测，再做极致优化；先本地私有证据，再声明性能达标。**

执行任何后续实现前，先读 `ACTIVE_GOAL.toml`、`perf/acceptance-matrix.toml`、`perf/current-loop-state.json`、`12_Review问题映射与修复责任.md`、`13_Loop_Engineering状态机.md`、`14_W0_W1验收矩阵与证据协议.md`、`15_性能观测与Profiling工具链.md`、`16_跨平台全盘发现与增量Journal契约.md` 和 `17_机器可读Goal与Experiment协议.md`。这些文档与 `python3 scripts/ci/check-performance-contracts.py` 一起构成防止目标漂移、benchmark 过拟合和证据混用的合同层。
