# 失败模式与恢复策略

## 1. 文件系统失败

| 失败 | 处理 | 用户可见 |
|---|---|---|
| watcher overflow | 标记 dirty subtree，重扫 | 显示正在补账 |
| FSEvents gap / USN gap | root 进入 reconciliation | 显示重新对账 |
| 文件被锁定 | retry_after，指数退避 | 显示稍后重试 |
| 外置盘断开 | root offline，不 tombstone 全部文件 | 显示 root 离线 |
| 云盘 partial download | 等待 stat 稳定 | 显示等待文件稳定 |
| 权限拒绝 | failed_retryable 或 failed_permanent | 显示权限问题 |
| macOS FSEvents gap | root 标记 dirty subtree，触发 bounded reconciliation | journal gap test + dirty subtree evidence |
| Windows USN gap | volume 标记 dirty subtree，触发 bounded reconciliation | journal gap test + dirty subtree evidence |
| watcher unavailable | fallback periodic manifest diff | degraded status + user-visible warning |
| symlink / reparse loop | 不跨出授权 root，记录 skipped alias | traversal loop test |
| cloud placeholder | 不强制下载，标记 partial/unavailable | cloud placeholder fixture |
| all-computer volume offline | root/volume offline，不批量 tombstone | offline volume reconciliation test |
| permission denied subtree | 跳过并记录 redacted count | permission aggregate evidence |

## 2. 存储失败

| 失败 | 处理 | 测试 |
|---|---|---|
| SQLite migration 失败 | fail closed，不继续写派生层 | migration red test |
| SQLite rollback journal 锁竞争或超时 | 读取保持不修改文件系统；发布写入在有界 busy timeout 内等待，失败则保留旧 projection/head | read-zero-mutation + bounded writer-wait focused meta-store tests |
| prepared batch 残留 | 启动时重放 | daemon restart test |
| fulltext 缺 batch | 从 ledger 补写 | index recovery test |
| sidecar checksum 失败 | snippet partial，不阻塞 search | sidecar corruption test |
| vector snapshot 损坏 | 禁用 semantic，保留 keyword/field | vector fallback test |

## 3. 查询失败

| 失败 | 处理 |
|---|---|
| 空 query | 返回提示或最近导入，不跑重路径 |
| 超长 query | 截断/摘要化并标记 warning |
| 互斥 filter | 返回空结果和原因 |
| ANN timeout | hybrid partial，BM25/field 仍返回 |
| snippet timeout | 返回无 snippet 结果 |
| hydrate timeout | 返回 partial page，不阻塞 daemon |

## 4. 后台任务失败

| 失败 | 处理 |
|---|---|
| OCR worker crash | job retry，达到上限后 failed_retryable |
| embedding command timeout | semantic partial，记录 model/runtime error |
| compaction 中断 | 旧 epoch 保留，重新排队 |
| kill daemon during import | 已 commit epoch 保留，prepared batch 重放 |
| disk full | pause background，front query read-only |
| low memory | 降级 budget profile，清 cache |

## 5. 失败可观测红线

任何失败不得只存在日志里。至少要进入：

1. daemon status。
2. redacted diagnostics。
3. GUI 可见状态。
4. benchmark/validation aggregate。

## 6. Contract Failure Modes

| 失败 | 恢复 | 验收 |
|---|---|---|
| interactive search overload | 返回 `OVERLOADED` 或 partial，不阻塞超过 deadline | IPC overload test |
| benchmark lane contamination | 标记 run invalid，不产生完整 baseline 声明 | benchmark summary rejects mixed lane |
| goal drift detected | 停止当前切片，回到 `ceo_reviewed` 或 `plan_ready` | Loop state evidence |
| repeated blocked evidence | 进入 `blocked`，报告外部输入需求 | blocked-stop report |
| raw private data in evidence | 当前目标失败，移除证据并重新生成 redacted aggregate | public repo guard + manual review |
| machine contract failure | 停止实现，修复 `ACTIVE_GOAL.toml`、`perf/*`、fixtures 或公开校验器 | `python3 scripts/ci/check-performance-contracts.py` |
| client disconnect/RST during any response | close and count only that connection; daemon/listener/workers remain alive | deterministic RST matrix plus next-request success |
| daemon child unexpectedly exits | native supervisor enters `retry_wait` and restarts within the in-process fixed budget/backoff | owned-child kill and lifecycle receipt |
| 5 automatic restarts in 10 monotonic minutes | open circuit for 5 monotonic minutes; allow one explicit half-open attempt; App restart gets a new budget | deterministic clock/budget test with no persistent ledger |
| configuration/protocol/integrity/ownership conflict | enter blocked; do not restart-loop or attach/kill another owner | duplicate-owner and corrupt-config tests |
| heartbeat transient failure | retain child until three consecutive 2-second heartbeat failures | injected heartbeat sequence |
| metadata initializing or blocked | authenticated status/diagnostics remain readable from memory; business routes return typed 503 without store access | delayed bootstrap and blocked-store injection |
| staging/snapshot/validation/CAS failure | old projection/readers remain fully readable; staged data stays invisible | failure injection at every publication step |
| target document changes active version | detail returns 409 `STALE_SELECTION`; UI clears private detail state | A-to-B immutable selection race |
| unrelated document advances epoch | exact still-active selection remains readable | multi-document epoch test |
| exact v29 manifest/key/integrity check fails | existing bytes remain unchanged and core becomes typed blocked | ciphertext hash and logical-preservation fixtures |
| v27/v28/unknown authority exists | return `unsupported_store_schema`; do not copy, migrate, delete, or initialize | negative hash-preservation fixtures |
| embedding unavailable | keyword/detail remain usable; semantic is unavailable; hybrid is lexical partial; vector-required publication stays frozen before claim | per-runtime capability matrix |
| OCR unavailable | text import remains available when otherwise allowed; scanned files remain unclaimed backlog | OCR claim-gate test |
| classifier unavailable | existing query generation remains readable; new import, reclassification, OCR publication, and index claim stay frozen | classifier epoch and claim-gate test |
| lifecycle bridge fails or status is invalid | revoke cached action authority and show `bridge_error` or `service_unknown` | strict validator and UI snapshot test |
