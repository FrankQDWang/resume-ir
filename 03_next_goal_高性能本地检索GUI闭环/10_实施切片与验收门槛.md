# 实施切片与验收门槛

## 1. 执行顺序

每个切片完成后单独验证、单独提交。

| 切片 | 目标 | 主要文件 |
|---|---|---|
| S1 | 目标文档和 contract baseline | `03_next_goal_高性能本地检索GUI闭环/`, `MANIFEST.md` |
| S2 | query artifact extractor 和 redacted summary | `crates/cli/src/main.rs`, `crates/cli/tests/s304_query_set.rs`, `scripts/ci/guard-public-repo.sh` |
| S3 | private query benchmark buckets 和 stage latency | `crates/benchmark-runner/src/lib.rs`, `crates/benchmark-runner/tests/s17_benchmark_runner.rs` |
| S4 | manifest/stable file identity/dirty subtree schema | `crates/meta-store/src/lib.rs`, `crates/meta-store/tests/s3_sqlite.rs`, `crates/fs-crawler/src/lib.rs` |
| S5 | mutation batch + visible epoch recovery | `crates/meta-store`, `crates/import-pipeline`, `crates/daemon` |
| S6 | fulltext fast-field/bitmap/bulk hydrate | `crates/index-fulltext`, `crates/search-planner`, `crates/rank-fusion` |
| S7 | snippet/body sidecar and daemon cache | `crates/index-fulltext`, `crates/daemon`, `crates/cli` |
| S8 | budget scheduler and OCR/semantic degradation | `crates/daemon`, `crates/ocr-client`, `crates/embedder`, `crates/index-vector` |
| S9 | performance validation script | `scripts/local/run-performance-goal-validation.sh`, `scripts/ci/check-performance-goal-validation.sh` |
| S10 | GUI/manual/Codex closed loop | GUI app files, daemon IPC tests, runbooks |
