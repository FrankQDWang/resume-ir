## 1. 执行顺序

每个切片完成后单独验证、单独提交。

| 切片 | 目标 | 主要文件 |
|---|---|---|
| P0 | goal authority、query semantics、IPC contract、acceptance matrix、loop state、public contract CI gate | `GOAL.md`, `.github/workflows/pr.yml`, `scripts/ci/check-performance-contracts.py`, `03_next_goal_高性能本地检索GUI闭环/`, `docs/superpowers/`, `perf/` |
| P1 | mixed import product-capability audit and frozen mixed benchmark contract | #138 audit, benchmark schemas, privacy docs, import contract docs |
| P1b | precision-first mixed import classifier | fs crawler source gate, import pipeline, meta-store audit/review records, classifier tests |
| P1c | recall/completeness recovery under precision guard | classifier rules, holdout evidence, review backlog metrics |
| P1d | mixed import L2 performance after semantics lock | parser workers, queues, backpressure, DB/index coordination, resource budgets |
| P2 | query semantics implementation and metamorphic checks | search planner, full-text parser, daemon search tests |
| P3 | snapshot/encryption ADR and storage mode decision | fulltext snapshot, storage docs, diagnostics |
| P4 | fulltext hot path: resident reader/writer, fast fields, bitsets, bulk hydrate | fulltext index, meta-store, daemon search |
| P5 | import first-searchable path, manifest diff, dirty subtree, stable identity | fs crawler, import pipeline, meta-store |
| P6 | platform journal strategy and reconciliation | platform-specific watcher crates, safe API boundary |
| P7 | adaptive governor and overload behavior | daemon scheduler, OCR/vector/import budgets |
| P8 | GUI visual contract, packaging, performance acceptance, and representative page | Tauri + React + Vite + Tailwind + TypeScript GUI app, daemon contract tests, manual checklist |
| P9 | final W1 benchmark, soak/fault, GUI/manual evidence | local validation, redacted reports |
| P10 | PGO/LTO/package-level tuning after semantics are locked | release/profile configuration |

No performance implementation slice may start before baseline and hypothesis are recorded in the linked GitHub profile issue. If no baseline exists, the only allowed work is instrumentation, benchmark harness, or baseline capture.
