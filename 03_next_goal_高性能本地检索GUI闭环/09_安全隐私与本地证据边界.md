# 安全隐私与本地证据边界

## 1. 数据分类

| 数据 | 是否可提交 | 说明 |
|---|---|---|
| synthetic fixtures | 可以 | 位于 `tests/fixtures/` |
| schema/version/contract | 可以 | 不含私有内容 |
| benchmark aggregate | 可以 | 只含计数、hash、percentile |
| local query set | 不可以 | 只留本机 |
| raw query text | 不可以 | 包括来自 artifacts 的 query |
| resume files | 不可以 | 包括文件名和路径 |
| SQLite/index/sidecar | 不可以 | 本地 runtime data |
| diagnostics raw package | 不可以 | 只能提交 redacted aggregate |
| model cache/runtime cache | 不可以 | 本地 runtime data |

| Material | Local read | Summarize | Hash | Commit | GitHub issue/PR |

GitHub issue/comment 只能写 redacted evidence。issue 是执行账本，不是私有数据存储位置。

## 2. SeekTalent Artifacts 边界

SeekTalent artifacts 只用于构建 query benchmark 的 query 形态。

允许输出到本仓库的内容：

1. 提取器代码。
2. schema。
3. 单元测试使用的 synthetic artifact fixture。
4. redacted aggregate summary。

不允许输出到本仓库的内容：

1. artifact 文件内容。
2. raw query。
3. query 对应结果。
4. candidate/card/resume/company/school/person text。
5. artifact 本机路径。

## 3. 脱敏检查

每个会产生证据的命令必须声明：

```json
{
  "contains_raw_resume_text": false,
  "contains_queries": false,
  "contains_resume_paths": false,
  "contains_candidate_results": false,
  "contains_artifact_paths": false
}
```

CI 必须 reject：

1. 私有路径。
2. 邮箱/电话/身份证样式。
3. raw query 字段。
4. candidate/result/card 字段。
5. SQLite/index/sidecar 文件。

## 4. 本地执行规则

真实私有 benchmark 只能由人类授权后在本机执行：

```bash
export RESUME_IR_PRIVATE_RESUME_ROOT="<local private resume root>"
export RESUME_IR_QUERY_ARTIFACT_ROOT="<local SeekTalent artifacts root>"
export RESUME_IR_LOCAL_EVIDENCE_DIR="<local private evidence output>"
```

提交前必须运行：

```bash
./scripts/ci/guard-public-repo.sh
```

## 5. Encryption Mode Boundary

索引加密策略必须先通过 ADR 冻结，不得在性能实现中隐式选择。

| Mode | 目标 | 约束 |
|---|---|---|
| performance mmap mode | 支持 mmap/streaming snapshot、低 RSS 热查询和快速恢复 | 只适合本机受信任存储；必须说明本机磁盘加密、权限和 threat model |
| strict app encrypted mode | 应用层加密派生索引、sidecar 和 diagnostics cache | 可能禁用 mmap 或降低性能；必须单独给出 benchmark、recovery 和 crash-safety 证据 |

默认产品模式必须由 P3 ADR 冻结。ADR 至少说明：

1. 默认模式。
2. key 来源和恢复策略。
3. crash-safety 与 partial write 恢复。
4. mmap、streaming snapshot 和 strict encryption 的性能取舍。
5. diagnostics cache 是否加密。
6. 备份、卸载、迁移和忘记 root 时的删除语义。
7. macOS/Windows 权限和磁盘加密假设。

当前文档默认不替 P3 做最终选择；实现前不得在代码中隐式选择 performance mmap mode 或 strict app encrypted mode。任何模式都不允许把 raw resume、raw query、path、token、trace 或 diagnostics package 提交到 git。

## 6. Anti-overfit 证据规则

1. local private query-set 可以留在本机，但 committed evidence 只能包含 hash、count、bucket、latency percentile 和 redacted aggregate。
2. 任何 raw query、候选结果、简历正文、姓名、联系方式、路径、token、trace 或截图进入 git，当前目标失败。
3. benchmark 至少区分 smoke、W0、W1、soak/fault、GUI/manual。不同 lane 的结果不得互相替代。
4. 查询语义是固定合同，不允许为了降低 latency 把 simple text AND 改成 OR、降低 hard filter 强度或跳过 partial explanation。
5. 性能报告必须记录 query set hash、sample count、bucket count、dataset count、searchable count、machine profile 和 percentile confidence。
6. W1 query set 必须固定 train/tune/holdout 或等价分层；调参只能看 tune，最终报告必须包含 holdout hash 和 bucket counts。
7. Metamorphic checks 必须覆盖 term reorder、add required term、explicit OR、field filter monotonicity 和 zero/partial explanation。
8. Drift policy 必须记录 query set version、extractor version、artifact input hash、dataset hash 和语义合同版本；任一变化都不能和旧 baseline 混报。

## 7. Machine Enforcement

当前 PR 的公开机器门槛：

1. `./scripts/ci/guard-public-repo.sh` 必须通过。
2. `python3 scripts/ci/check-performance-contracts.py` 必须通过。
3. `.github/workflows/pr.yml` 必须运行 Performance contract check。
4. `ACTIVE_GOAL.toml`、`perf/acceptance-matrix.toml`、`perf/current-loop-state.json`、`perf/loop-state.schema.json` 和 `perf/experiment-report.schema.json` 必须能被 parser 读取并满足 v2 合同。
5. `perf/fixtures/valid/*.json` 必须全部通过，`perf/fixtures/invalid/*.json` 必须全部失败。
6. review ledger 不能缺少 `status`、`closure_evidence` 和 `closed_by`。
7. D10K/D100K fixture 不能声明 `goal_complete`；空 W1、空 commands、缺失 accepted cells 必须失败。

后续实现阶段的机器门槛：

1. private query extractor 必须默认输出 local-only query set 和 redacted summary 两个分离文件。
2. redacted summary 必须通过 `perf/experiment-report.schema.json` 的隐私字段。
3. W1 报告中任一 `contains_*` 隐私字段为 true，当前目标失败。
4. holdout 结果缺失时，不能声明 W1 accepted。
5. 调参后的报告如果没有 query semantics version 和 matrix version，不能与旧报告比较。
