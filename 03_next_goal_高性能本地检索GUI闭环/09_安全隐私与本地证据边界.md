# 安全隐私与本地证据边界

## 1. 数据分类

| 数据 | 是否可提交 | 说明 |
|---|---|---|
| synthetic fixtures | 可以 | 位于 `tests/fixtures/` |
| schema/version/contract | 可以 | 不含私有内容 |
| benchmark aggregate | 可以 | 只含计数、hash、percentile |
| local query set | 不可以 | 只留本机 |
| raw query text | 不可以 | 包括来自 artifacts 的 query |
| resume files | 不可以 | 包括文件名和路径 |
| SQLite/index/sidecar | 不可以 | 本地 runtime data |
| diagnostics raw package | 不可以 | 只能提交 redacted aggregate |
| model cache/runtime cache | 不可以 | 本地 runtime data |

## 2. SeekTalent Artifacts 边界

SeekTalent artifacts 只用于构建 query benchmark 的 query 形态。

允许输出到本仓库的内容：

1. 提取器代码。
2. schema。
3. 单元测试使用的 synthetic artifact fixture。
4. redacted aggregate summary。

不允许输出到本仓库的内容：

1. artifact 文件内容。
2. raw query。
3. query 对应结果。
4. candidate/card/resume/company/school/person text。
5. artifact 本机路径。

## 3. 脱敏检查

每个会产生证据的命令必须声明：

```json
{
  "contains_raw_resume_text": false,
  "contains_queries": false,
  "contains_resume_paths": false,
  "contains_candidate_results": false,
  "contains_artifact_paths": false
}
```

CI 必须 reject：

1. 私有路径。
2. 邮箱/电话/身份证样式。
3. raw query 字段。
4. candidate/result/card 字段。
5. SQLite/index/sidecar 文件。

## 4. 本地执行规则

真实私有 benchmark 只能由人类授权后在本机执行：

```bash
export RESUME_IR_PRIVATE_RESUME_ROOT="<local private resume root>"
export RESUME_IR_QUERY_ARTIFACT_ROOT="<local SeekTalent artifacts root>"
export RESUME_IR_LOCAL_EVIDENCE_DIR="<local private evidence output>"
```

提交前必须运行：

```bash
./scripts/ci/guard-public-repo.sh
```
