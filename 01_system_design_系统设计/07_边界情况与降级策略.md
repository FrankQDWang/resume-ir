# 边界情况与降级策略

## 1. 硬件分档

| 档位 | 典型机器 | 默认策略 |
|---|---|---|
| Economy | 4 核以下、8GB 内存、机械盘或资源紧张 | 关闭实时 OCR，向量延迟，索引小批量提交 |
| Balanced | 4-8 核、16GB 内存、SSD | 默认策略，OCR 限流，向量后台生成 |
| Turbo | 8 核以上、32GB+、高速 SSD | 批量导入加速，多任务并行 |

系统启动后应自动检测资源，并允许用户手动切换。

## 2. 功能降级顺序

当资源不足时，按以下顺序降级：

1. 暂停 OCR。
2. 暂停向量生成。
3. 暂停索引合并。
4. 降低导入并发。
5. 关闭 snippet 缓存。
6. 降低向量 topK。
7. 语义检索返回 partial。
8. 保留文件名和结构化基础查询。

查询必须优先于后台任务。

## 3. 文件边界情况

| 情况 | 策略 |
|---|---|
| 路径过长 | 使用规范化 URI，不依赖展示路径作为唯一 ID |
| 文件名乱码 | 保留原始字节名和可显示名 |
| 同名文件 | 以路径和内容指纹区分 |
| 文件移动 | 内容 hash 一致则视为移动 |
| 文件被同步软件反复改写 | 事件合并，延迟处理 |
| 文件被杀毒软件锁定 | 延迟重试 |
| 外接盘断开 | 标记源不可达，不立即删除 |
| 权限不足 | 记录不可访问，不反复重试 |

## 4. 文档格式边界情况

| 情况 | 策略 |
|---|---|
| 加密 PDF | 不暴力破解，标记需要密码 |
| 损坏 PDF | 尝试轻量修复，失败后记录 |
| 图片简历 | 进入 OCR，但限页、限时 |
| 双栏 PDF | 保留版面位置信息，尝试顺序修正 |
| 表格型 Word | 表格线性化，保留邻近关系 |
| Word 内嵌扫描图片 | 文本不足时对图片 OCR |
| 页眉页脚重复 | 清洗阶段去重 |
| 证件照 | 不做人脸识别，不入索引 |
| 超长简历 | 分段截断 + 全文保留，向量只取关键段 |

## 5. 解析质量边界

字段抽取不可避免会有误差。必须设计为可解释：

1. 每个字段有原文证据。
2. 每个字段有置信度。
3. 每个字段知道来源：规则、词典、模型、人工。
4. 冲突字段不直接覆盖，保留候选。
5. 搜索过滤默认只使用高置信字段。

## 6. OCR 边界

| 情况 | 策略 |
|---|---|
| 图片倾斜 | 轻量方向检测和纠偏 |
| 低分辨率 | 降低置信度，不强抽字段 |
| 印章/水印干扰 | 清洗阶段过滤明显噪音 |
| 中英混排 | 多语言 OCR profile |
| OCR 结果乱码 | 标记低质量，不进入强字段过滤 |
| 单文件 OCR 过慢 | 页级超时，保留部分结果 |

## 7. 查询边界

| 查询类型 | 策略 |
|---|---|
| 空查询 | 返回最近导入或提示补充条件 |
| 超长查询 | 摘取关键词和语义摘要，限制长度 |
| 多个互斥过滤条件 | 返回空结果和原因解释 |
| 模糊地区/学校 | 使用别名表和同义映射 |
| 错别字 | 文件名和技能词可做轻量纠错 |
| 大 topK | 使用分页游标 |
| 查询超时 | 返回 partial + 超时标记 |

## 8. 数据一致性边界

| 情况 | 策略 |
|---|---|
| 元数据已写，索引未写 | 状态为 partial，不作为完整结果 |
| 倒排已写，向量未写 | 可全文检索，语义检索 partial |
| 向量索引损坏 | 禁用语义，保留全文 |
| 元数据损坏 | 从最近备份恢复或重建 |
| schema 版本变化 | 迁移，不兼容则重建相关索引 |
| 应用崩溃 | 未完成任务重放 |

## 9. 用户体验边界

低配机器上必须明确告诉用户系统在做什么：

1. 当前可搜索数量。
2. 待 OCR 数量。
3. 待向量生成数量。
4. 后台任务是否暂停。
5. 为什么暂停。
6. 如何切换性能档位。

不要让用户误以为“导入完成 = 所有 OCR 和语义索引都完成”。
