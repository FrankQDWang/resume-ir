# 输入输出与接口契约

## 1. 输入来源

| 来源 | 示例 | 处理方式 |
|---|---|---|
| 监听目录 | `D:\resumes`、`~/Documents/resumes` | 增量扫描 + 文件变更监听 |
| 一次性导入 | 用户拖拽文件夹 | 批量任务 |
| 单文件导入 | 一个 Word/PDF | 高优先级任务 |
| 迁移导入 | 另一台机器的索引包 | 校验版本后导入 |
| agent 调用 | 本地接口传入路径或查询 | 权限校验后执行 |

## 2. 导入配置契约

```yaml
import_job:
  source_id: "default-local-folder"
  roots:
    - "/Users/me/resumes"
    - "D:/resumes"
  include_ext: ["docx", "pdf", "doc", "txt"]
  exclude_patterns:
    - "**/~$*"
    - "**/.DS_Store"
    - "**/node_modules/**"
  follow_symlink: false
  ocr:
    enabled: true
    mode: "balanced"       # off/economy/balanced/turbo
    languages: ["chi_sim", "eng"]
    max_pages_per_doc: 20
  privacy:
    hash_contact_fields: true
    store_raw_text: true
  priority: "normal"
```

## 6. 错误码

| 错误码 | 含义 | 用户提示 |
|---|---|---|
| `FILE_NOT_FOUND` | 文件不存在 | 文件可能被移动或删除 |
| `FILE_LOCKED` | 文件被占用 | 稍后自动重试 |
| `UNSUPPORTED_FORMAT` | 不支持格式 | 仅记录元数据 |
| `ENCRYPTED_DOCUMENT` | 文档加密 | 需要密码或跳过 |
| `CORRUPTED_DOCUMENT` | 文档损坏 | 可尝试重新导出 |
| `OCR_TIMEOUT` | OCR 超时 | 降级为部分可搜索 |
| `PARSER_TIMEOUT` | 解析超时 | 后台重试或跳过 |
| `LOW_MEMORY_PAUSED` | 内存不足暂停 | 等待资源恢复 |
| `INDEX_SNAPSHOT_INVALID` | 快照校验失败 | 回滚到上一快照 |
| `SCHEMA_MISMATCH` | 索引 schema 不兼容 | 触发迁移或重建 |

## 7. CLI 契约

```bash
resume-engine import --root <path> --profile balanced
resume-engine search "Java 支付" --degree bachelor --top-k 20
resume-engine status
resume-engine pause --task ocr
resume-engine resume --task ocr
resume-engine rebuild --scope vector
resume-engine doctor
resume-engine export-diagnostics --redact
```

## 8. 本地 agent 调用边界

agent 可以调用：

1. 搜索。
2. 获取候选简历结构化字段。
3. 获取某个文档的片段。
4. 提交导入任务。
5. 查询索引状态。
6. 暂停或恢复重任务。

agent 不应直接读写内部索引文件。

## 9. 分页与大结果集

所有大结果集必须分页或流式返回。

规则：

1. 默认 `top_k <= 100`。
2. 超过 100 使用游标。
3. 不一次性返回全文。
4. 原文获取必须单独调用，并默认脱敏。
5. 导出功能必须有审计记录。
