# 设计原则与需求边界

## 1. 目标场景

系统用于在用户电脑本地管理和检索大规模简历库。简历可能来自文件夹、压缩包解压目录、招聘系统导出目录、人工拖拽目录或第三方同步目录。

必须支持：

| 类别 | 要求 |
|---|---|
| 文件格式 | `.docx`、`.pdf`、文本层 PDF、扫描 PDF、图片型 PDF；旧 `.doc` 作为兼容项处理 |
| 数据规模 | 10 万到 100 万为常规目标，300 万作为扩展目标 |
| 运行环境 | Windows 与 macOS 桌面环境；机器性能差异很大 |
| 使用模式 | 本地软件、后台守护进程、另一个 agent 调用、本地 CLI 或 UI 调用 |
| 检索类型 | 文件名、全文、结构化字段、语义、混合检索 |
| 数据安全 | 简历属于敏感个人信息，默认不出本机 |

## 2. 非目标

以下内容不是本系统第一优先级：

1. 不做云端搜索服务。
2. 不把查询请求发送到公网模型或公网搜索服务。
3. 不要求多用户共享索引。
4. 不要求局域网多人协作。
5. 不在查询时做 OCR、文档解析、全量重排或大模型抽取。
6. 不保证所有损坏、加密、超复杂版式文件都能 100% 解析，只保证有状态、有错误码、有可恢复路径。

## 3. 性能口径

性能目标必须按链路拆分，不能只写一个“快”。

| 链路 | 目标 | 说明 |
|---|---:|---|
| 文件名/精确字段查询 | P95 `<50ms` | 热索引、无大结果集传输 |
| 全文关键词查询 | P95 `<100ms` | topK 返回，不含首次冷启动 |
| 语义查询 | P95 `<150ms` | 向量索引已加载或可 mmap |
| 混合查询 | P95 `<200ms` | 倒排 + 向量 + 融合 + topN snippet |
| 单文件导入可见性 | 秒级到分钟级 | 取决于是否 OCR |
| 批量导入 | 后台长期运行 | 不阻塞查询 |

这些是工程目标，不是无条件承诺。低配机器、扫描件占比高、机械硬盘、杀毒软件扫描、外接移动盘都会显著影响导入速度。

## 4. 设计原则

### 4.1 查询热路径只读

查询路径只允许做：

1. 查询解析。
2. 字段过滤规划。
3. 倒排召回。
4. 向量召回。
5. 候选融合。
6. 去重和版本折叠。
7. 片段读取和高亮。
8. 返回结果。

查询路径禁止做：

1. OCR。
2. 文档格式解析。
3. 重新分段。
4. 全量字段抽取。
5. 全量向量生成。
6. 大模型长文本推理。
7. 大规模索引合并。

### 4.2 摄取链路异步

导入、解析、OCR、字段抽取、向量生成、索引写入全部走后台任务队列。

用户查询时可以看到“部分可搜索”状态：

| 状态 | 可搜索内容 |
|---|---|
| 仅发现文件 | 文件名、路径、大小、修改时间 |
| 已抽取文本 | 文件名、全文 |
| 已抽取字段 | 文件名、全文、结构化字段 |
| 已生成向量 | 文件名、全文、字段、语义 |
| 已完成 OCR | 扫描 PDF 的 OCR 文本和字段 |

### 4.3 分层索引

系统必须把数据分成不同用途的索引层：

| 层 | 用途 | 热度 |
|---|---|---|
| 元数据层 | 文件、任务、版本、状态、错误、权限、本地配置 | 热 |
| 倒排层 | 文件名、全文、短语、布尔查询、字段过滤 | 热 |
| 字段列层 | 年限、学历等级、地区、时间、数值范围、枚举 | 热 |
| 向量层 | 语义召回、相似简历、相似岗位匹配 | 温 |
| 缓存层 | OCR 结果、片段、高频 query、embedding 中间结果 | 温/冷 |
| 原始文件层 | 原始 Word/PDF 文件或路径引用 | 冷 |

### 4.4 快照化与可恢复

索引写入和索引读取必须隔离。后台写入新 segment 或新快照，查询线程只读取已提交快照。

崩溃恢复口径：

1. 已提交快照必须可读。
2. 未提交任务可以重试。
3. 任务状态必须可追踪。
4. 文件删除必须能传播到索引删除标记。
5. 旧 segment 回收必须延迟到没有读者使用。

### 4.5 低配机器优先稳定

低配机器上，宁可牺牲一些召回质量，也不能让电脑卡死。

资源调度优先级：

1. 查询响应。
2. UI/agent 交互。
3. 轻量导入。
4. 字段抽取。
5. 向量生成。
6. OCR。
7. 索引合并。
8. 质量重算和全量重建。
